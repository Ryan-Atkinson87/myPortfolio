Most robotics projects start with a kit and end with a kit. Robert started with a question from my son: “Can we build a robot that we actually control?” So we did.

Robert is a physical robot (wheels, a four-servo arm, a claw) on a Raspberry Pi, controlled entirely through a browser. No app to install, no proprietary remote. Just a web page sending commands to real hardware.

It exists for two reasons: to build something genuinely fun and useful with my son, where his input shapes what Robert does next, and to explore the full stack of a hardware-connected web app, from GPIO and I2C servo control at the bottom, to a Flask API in the middle, to a browser UI at the top.
